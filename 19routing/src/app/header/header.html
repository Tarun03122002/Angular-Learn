routerLink directive is a directive that binds HTML element to a route.It catches click event on html element and it result in navigating to that route

href="home" will refresh the application and its' state,so we will not use this

routerLinkActive directive is used to add or remove class from html element that is bound to RouterLink.It's use case is to show highlight the currently active route

Note : when a child route is active ,then all the parent routes are also marked as active,in that case routerLink is applied to active child route and all its parent route

routerLinkActiveOptions directive pass extra paramters exact to inform routerLinkActive how to match route for styling active route

Route parameter is mandatory,without this route will not work,whereas query parameter is optional,Like in home screen ,if we pass query param ,it will redirect to course on clicking search, if we will not pass query string,home component view will be rendered.

Absolute vs relative path

When using directive routerLink if "/" is used it means absolute path,route is appended to root url,if "/" is not used,it means relative path,route is appended to currently active route

if we use dot and slash("./"),it will become relative path,
Eg. localhost:4200/home/courses -> currently active route
-> it will navigate to localhost:4200/home/courses/course1

if we use("../"),it will move one level up,path will be appended to parent path
Eg. localhost:4200/home/courses -> currently active route
-> it will navigate to localhost:4200/home/dashboard

if we use("../../"),it will move two level up,path will be appended after removing last two paths of parent url
Eg. localhost:4200/books/author -> currently active route
-> it will navigate to localhost:4200s/tephen
